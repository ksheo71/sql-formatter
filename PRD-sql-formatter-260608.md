# PRD: SQL Formatter (웹)

> **한 줄 정의**: 브라우저에서 SQL 코드를 붙여넣으면 sqlinform 수준의 세밀한 옵션으로 즉시 정렬·포맷팅해주는 무료 개발자 도구
> **작성일**: 2026-06-08
> **버전**: v1.0
> **타겟 바이브코딩 환경**: Claude Code
> **작성자**: kyoungseok.heo

---

## 1. 프로젝트 정체성

- **한 줄 정의**: 브라우저에서 SQL을 붙여넣고 옵션을 고르면 일관된 스타일로 즉시 포맷팅해주는 클라이언트 사이드 웹 도구.
- **비전 / 미션**: "SQL 스타일 정리에 더 이상 시간을 쓰지 않는다." 팀/개인이 합의한 SQL 코드 스타일을 한 번 설정해두고 어디서든(웹 → 향후 VSCode) 동일하게 적용한다.
- **핵심 가치 제안 (Value Proposition)**:
  - 가입·설치 없이 즉시 사용 (붙여넣기 → 포맷 → 복사).
  - sqlinform 수준의 **세밀한 옵션 제어** (키워드 케이스, 들여쓰기, 콤마 위치, 정렬 등).
  - 모든 처리를 **브라우저 내(client-side)** 에서 수행 → SQL이 서버로 전송되지 않아 보안 우려 없음.
  - 포맷팅 로직을 코어 패키지로 분리해 **향후 VSCode 확장에서 동일 엔진 재사용**.
- **이해관계자**: 단독 개발/운영자(본인), 사용자(개발자/데이터 분석가), 외부 OSS 의존성(sql-formatter 라이브러리).

## 2. 문제 정의

- **해결하려는 문제**: 개발자/분석가가 작성하거나 전달받은 SQL은 들여쓰기·대소문자·콤마 위치가 제각각이라 가독성이 떨어지고 리뷰가 어렵다. 매번 수작업 정리는 비효율적이다.
- **현재 솔루션의 한계**:
  - 온라인 무료 포맷터 다수는 옵션이 빈약하거나(케이스/들여쓰기 정도) 광고가 과하다.
  - 옵션이 풍부한 도구(예: sqlinform)는 데스크톱/유료 위주이거나 UI가 무겁다.
  - 일부 도구는 SQL을 서버로 전송해 민감 쿼리 노출 우려가 있다.
- **차별화 포인트 (Why now? Why us?)**: 풍부한 옵션 + 클라이언트 사이드 처리(보안) + 무가입 즉시성 + 향후 VSCode 확장으로 동일 스타일 이식이라는 조합. 검증된 OSS 엔진 위에 sqlinform식 옵션 UX를 얹어 빠르게 출시.

## 3. 타겟 사용자

- **핵심 페르소나**: 백엔드/데이터 엔지니어, 데이터 분석가(20~40대, 디지털 친숙도 상). 매일 SQL을 작성·리뷰하며, 빠르게 "붙여넣고 정리"하길 원함. 회사 PC 브라우저에서 사용.
- **부 페르소나**: SQL을 가끔 다루는 기획자/PM, SQL 학습자(가독성 좋은 예시 정리 목적).
- **안티-페르소나**: SQL을 다루지 않는 일반 사용자, 엔터프라이즈 온프레미스 배포/SSO를 요구하는 조직(MVP 비대상).

## 4. 핵심 사용자 시나리오

### 4.1 시나리오 1 (우선순위 1): 즉시 포맷팅
- **Given**: 사용자가 정리되지 않은 SQL 한 덩어리를 클립보드에 갖고 사이트에 접속한다.
- **When**: 좌측 입력 에디터에 SQL을 붙여넣는다.
- **Then**: 별도 버튼 클릭 없이(또는 1클릭으로) 우측에 기본 옵션으로 포맷팅된 결과가 표시되고, "복사" 버튼으로 즉시 클립보드에 담을 수 있다.

### 4.2 시나리오 2 (우선순위 2): 옵션 커스터마이즈
- **Given**: 사용자가 키워드는 대문자, 들여쓰기 2칸, 콤마는 줄 앞(leading comma)을 원한다.
- **When**: 옵션 패널에서 해당 항목을 변경한다.
- **Then**: 결과가 실시간(또는 즉시 재포맷)으로 반영되고, 선택한 옵션은 브라우저에 저장되어 다음 방문 시 유지된다.

### 4.3 시나리오 3 (우선순위 3): 방언(dialect) 전환
- **Given**: 사용자가 PostgreSQL 전용 문법(예: `::` 캐스팅)이 포함된 쿼리를 다룬다.
- **When**: 방언 셀렉터에서 PostgreSQL을 선택한다.
- **Then**: 해당 방언 규칙에 맞게 파싱·포맷팅되어 문법이 깨지지 않는다.

### 4.4 북극성 지표
- **북극성 지표**: "방문 후 1분 내 첫 포맷팅 성공률" (붙여넣기 → 유효한 포맷 결과 → 복사 도달 비율). [가정] — 무가입 도구 특성상 첫 핵심 가치 체감 속도를 핵심 지표로 설정.
- 보조 지표: 재방문율(옵션 저장 활용), 평균 포맷팅 횟수/세션.

## 5. 기능 요구사항

### 5.1 MVP 필수 기능 (Must)

| ID | 기능명 | 설명 | 인수조건 |
| -- | -- | -- | -- |
| F-01 | SQL 입력/출력 듀얼 에디터 | 좌(입력)·우(출력) 분할 에디터, SQL 구문 강조 | Given 입력 에디터에 SQL을 붙여넣으면 When 포맷 실행 시 Then 우측에 구문 강조된 결과가 표시된다 |
| F-02 | 즉시 포맷팅 | 붙여넣기/입력 변경 시 자동 또는 1클릭 포맷 | Given 유효한 SQL이 입력되면 When 자동 트리거(디바운스) 또는 'Format' 클릭 시 Then 0.5초 내 결과가 갱신된다 |
| F-03 | 결과 복사 | 포맷 결과를 클립보드로 복사 | Given 결과가 존재하면 When '복사' 클릭 시 Then 클립보드에 전체 텍스트가 담기고 완료 토스트가 뜬다 |
| F-04 | SQL 방언 선택 | 표준 SQL + 주요 방언 지원 | Given 방언 셀렉터에서 항목 선택 When 포맷 실행 시 Then 해당 방언 규칙으로 파싱·포맷된다 (지원 목록: 7.3 참조) |
| F-05 | 포맷 옵션 패널 (sqlinform식 카테고리 UI) | 키워드 케이스, 들여쓰기, 콤마 위치, 정렬, 공백 등 옵션을 카테고리별로 제공 | Given 옵션을 변경하면 When 재포맷 시 Then 변경이 결과에 반영된다 (옵션 목록: 5.4 참조) |
| F-06 | 옵션 로컬 저장 | 선택 옵션을 브라우저 localStorage에 자동 저장 | Given 옵션을 변경하고 페이지를 새로고침하면 Then 직전 옵션이 그대로 복원된다 |
| F-07 | 입력 비우기/초기화 | 입력·옵션 초기화 버튼 | Given 'Clear' 클릭 시 Then 입력이 비워지고, 'Reset options' 클릭 시 Then 옵션이 기본값으로 돌아간다 |
| F-08 | 에러/미지원 처리 | 파싱 불가·미지원 구문 시 명확한 안내 | Given 파싱 실패 SQL When 포맷 시도 시 Then 원문을 훼손하지 않고 에러 메시지를 표시한다 |
| F-09 | 반응형 레이아웃 | 데스크톱 좌우 분할, 모바일 상하 분할 | Given 모바일 폭에서 접속 When 화면 표시 시 Then 입력/출력이 상하로 재배치되어 사용 가능하다 |

- O-01: **서버 저장 / 포맷 히스토리** — 모든 처리는 클라이언트(브라우저)에서만. 백엔드 DB·서버 측 SQL 저장 없음. *(사용자 명시 선택)*
- O-02: **회원가입 / 로그인 / 계정 시스템** — 인증 없이 누구나 즉시 사용하는 공개 도구. ✅ 사용자 확인(C-03)
- O-03: **AI/LLM 기반 포맷팅** — 규칙 기반 파서/포맷터만 사용, LLM API 호출 없음. ✅ 사용자 확인(C-03)
- O-04: **결제 / 유료화 / 구독** — MVP는 전부 무료. ✅ 사용자 확인(C-03)
- O-05: **다국어 UI** — MVP는 영어 또는 한국어 단일 UI(8.2 참조). ✅ 사용자 확인(C-03)
- O-06: **SQL 실행 / DB 연결 / 쿼리 결과 조회** — 포맷팅 전용. SQL을 실제로 실행하지 않음.
- O-07: **SQL 문법 검증/린팅(스타일 외)** — 의미적 검증·성능 분석은 대상 아님.

## 6. 데이터 모델

> O-01에 따라 **서버 측 데이터 모델 없음**. 상태는 전부 클라이언트(메모리 + localStorage).

### 6.1 엔티티 관계 (간이 ERD - 텍스트)
```
(브라우저 localStorage)
FormatOptions 1 ─── 1 LastSession(입력 텍스트는 저장 안 함, 옵션만 영속)
```

### 6.2 엔티티별 필드 (클라이언트 상태)

**FormatOptions** (localStorage 키: `sqlfmt.options`)
| 필드 | 타입 | 필수 | 설명 |
| -- | -- | -- | -- |
| dialect | string(enum) | Y | 선택된 SQL 방언 |
| keywordCase | enum(upper/lower/preserve) | Y | 키워드 케이스 |
| identifierCase | enum | Y | 식별자 케이스 |
| tabWidth | number | Y | 들여쓰기 공백 수 (기본 2) |
| useTabs | boolean | Y | 탭 사용 여부 |
| indentStyle | enum | Y | 들여쓰기 스타일 |
| commaPosition | enum(trailing/leading) | Y | 콤마 위치 |
| logicalOperatorNewline | enum(before/after) | Y | 논리 연산자 줄바꿈 |
| expressionWidth | number | N | 표현식 최대 줄폭 |
| linesBetweenQueries | number | N | 쿼리 간 빈 줄 |
| denseOperators | boolean | N | 연산자 밀집 |
| newlineBeforeSemicolon | boolean | N | 세미콜론 앞 줄바꿈 |
| version | number | Y | 스키마 버전(마이그레이션용) |

**EditorState** (런타임 메모리, 비영속)
| 필드 | 타입 | 설명 |
| input | string | 입력 SQL (저장 안 함, O-01) |
| output | string | 포맷 결과 |
| error | string \| null | 파싱 에러 메시지 |

### 6.3 외부 데이터 소스
- 없음. 모든 포맷팅은 번들된 클라이언트 라이브러리로 수행. 외부 API 호출 없음.

- **트레이드오프**: 라이브러리가 sqlinform의 23개 카테고리를 100% 커버하지는 못함 → 미지원 고급 옵션은 Phase 2 후처리기(N-05)로 단계 구현. 이 격차는 13.2 리스크·13.3 [확인필요]에 기록.

### 7.3 지원 SQL 방언 (MVP)

## 9. 비기능 요구사항

- **성능**: 일반 길이(수백 줄) SQL 포맷팅 < 0.5초(클라이언트). 초기 로드(에디터+엔진 번들) 합리적 코드 스플리팅.
- **보안/프라이버시**: SQL이 서버로 전송되지 않음(전 클라이언트 처리). 입력은 영속 저장 안 함. `/about`에 명시 고지.
- **접근성**: 키보드 내비게이션, 적절한 대비, 버튼 aria-label.
- **반응형**: 데스크톱 좌우 분할 / 모바일 상하 분할(F-09). 반응형 웹만, 네이티브 앱 빌드 없음 [가정].
- **브라우저 지원**: 최신 Chrome/Edge/Safari/Firefox.
- **다국어**: O-05에 따라 단일 언어.

## 12. 인수 시나리오 (Acceptance Scenarios)

- **AC-01 (F-01/F-02/F-03)**: Given 정리 안 된 SELECT 쿼리를 입력에 붙여넣는다 When 자동/클릭 포맷 시 Then 우측에 들여쓰기·줄바꿈 정리된 결과가 구문 강조되어 나타나고, '복사' 시 클립보드에 동일 텍스트가 담긴다.
- **AC-02 (F-05/F-06)**: Given 키워드 케이스를 UPPER, 콤마를 leading으로 변경 When 재포맷 후 새로고침 Then 결과에 옵션이 반영되며, 새로고침 후에도 동일 옵션이 복원된다.
- **AC-03 (F-04)**: Given PostgreSQL `::` 캐스팅 포함 쿼리, 방언=PostgreSQL When 포맷 시 Then 문법이 깨지지 않고 정상 포맷된다.
- **AC-04 (F-08)**: Given 파싱 불가능한 깨진 SQL When 포맷 시도 Then 원문이 사라지지 않고 명확한 에러 메시지가 표시된다.
- **AC-05 (F-09)**: Given 모바일 폭(<768px)으로 접속 When 화면 표시 Then 입력/출력이 상하로 재배치되어 사용 가능하다.
- **AC-06 (O-01 검증)**: Given 임의 SQL 입력 When 네트워크 탭 관찰 Then 입력 SQL이 외부로 전송되는 요청이 없다.
